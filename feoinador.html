CSS

CSS (Cascading Style Sheets) is a stylesheet language used to describe the visual presentation of a document written in HTML or XML. While HTML handles the structure and content of a page, CSS controls how that content looks — its colors, fonts, spacing, layout, and more. The main purpose of CSS is to separate content from presentation. Before CSS, styling had to be written directly inside HTML tags, making code messy and hard to maintain.

There are 3 ways to apply CSS styles to an HTML document:

  1. Inline CSS: Written directly inside an HTML tag using the style attribute. It applies only to that specific element. Useful for quick one-off overrides, but hard to maintain at scale.
  2. Internal CSS: Written inside a style tag in the head section of an HTML file. It applies to the whole page. Good for single-page projects or page-specific styles.
  3. Extenal CSS (Recomendado): Written in a separate .css file and linked to the HTML using a link tag. This is the best practice — it keeps styles separate from content and can be reused across many pages.

CSS Selector and Rule

A CSS rule is the complete block of code that tells the browser how to style specific elements. It consists of a selector and a declaration block. A selector is the part of the rule that targets the HTML element(s) you want to style. It tells CSS: "apply these styles to THIS element." <W3Schools.com. (s. f.-d). https://www.w3schools.com/css/css_selectors.asp>


Types of Selectors

A class is an attribute you can add to any HTML element to group it with other elements that share the same styling needs. Classes are reusable — you can apply the same class to as many elements as you want, and one element can have multiple classes separated by spaces.

An ID is an attribute used to identify a single, unique element on a page. No two elements should share the same ID. IDs are commonly used for page anchors, JavaScript hooks, and high-specificity CSS rules.

  1. Tag Selector: targets all HTML elements of a given type on the page. You simply write the tag name without any prefix.
  2. Class Selector: targets any element that has a specific class attribute. It is written with a dot before the class name. Multiple elements can share the same class, and one element can have multiple classes.
  3. Tag and Class Selector: You can combine a tag name and a class name together (no space between them) to target only elements of that specific tag that also have that specific class. This is more precise than using the class alone.
  4. ID Selector: targets a single, unique element on the page identified by its id attribute. It is written with a hash ( # ) before the ID name. An ID must be unique — no two elements on the same page should share the same ID.

CSS Box Model

Every HTML element is treated as a rectangular box. The CSS Box Model describes the four layers that make up this box, from the inside out:

  1. Content: The innermost layer where text and images appear.
  2. Padding: The space between the content and the border. It creates breathing room inside the box.
  3. Border: A line that wraps around the padding and content. It can be styled with different widths, colors, and patterns.
  4. Margin: The outermost layer that creates space between this element and other elements on the page. Margins are transparent and do not have a background color.

Media Queries

Media queries are a CSS feature that lets you apply styles conditionally based on characteristics of the device or viewport, most commonly the screen width. They are the foundation of responsive web design, allowing a single HTML file to look great on phones, tablets, and desktops.
W3Schools.com. (s. f.-e). https://www.w3schools.com/css/css_rwd_mediaqueries.asp

CSS Displays

Display: Grid

CSS Grid is a two-dimensional layout system. It lets you arrange elements into rows AND columns at the same time, making it ideal for building full page layouts, dashboards, and any design that needs structure in both directions.

Display: Flex

CSS Flexbox (Flexible Box Layout) is a one-dimensional layout system. It arranges items either in a row or a column, but not both at the same time like Grid. It is ideal for navigation bars, centering content, aligning buttons, and distributing space among items in a single direction.
